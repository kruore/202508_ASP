#동시성

서버는 수천의 처리를 한번에 하기 위해서
스레드를 생성하게 된다.

Shared_ptr이나 전역 변수처럼, 여러 스레드가 하나의 메모리 공간을 공유할 때 문제 발생

동시에 처리하게 되면 값이 어떻게 처리되는가?
CPU는 **[Load -> Modify -> Store]라는 과정을 거치게 된다.

스레드 B가 스레드 A의 'Store' 작업이 끝나기도 전에(즉, 업데이트되지 않은 **옛날 값**을) 'Load' 해버리는 것이 원인으로 발생

이를 제어하기 위해서 #동기화 Synchronization이 필요하다.
